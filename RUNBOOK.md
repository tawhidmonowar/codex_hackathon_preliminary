# RUNBOOK — QueueStorm Investigator

## Quick Run (Local)

```bash
# 1. Install dependencies
composer install

# 2. Copy environment
cp .env.example .env

# 3. Set your Gemini API key
# Edit .env → GEMINI_API_KEY=your_key_here

# 4. Generate application key
php artisan key:generate

# 5. Create SQLite database and run migrations
touch database/database.sqlite
php artisan migrate

# 6. Start server
php artisan serve --host=0.0.0.0 --port=8000
```

## Docker Run

```bash
# Build
docker build -t queuestorm .

# Run (replace YOUR_KEY with actual Gemini API key)
docker run -p 8000:8000 -e GEMINI_API_KEY=YOUR_KEY queuestorm
```

## Verify

```bash
# Health check
curl http://localhost:8000/health
# Expected: {"status":"ok"}

# Analyze ticket
curl -X POST http://localhost:8000/analyze-ticket \
  -H "Content-Type: application/json" \
  -d '{"ticket_id":"TKT-TEST","complaint":"Test complaint","language":"en"}'
```

## Environment Variables

| Variable | Required | Default |
|----------|----------|---------|
| GEMINI_API_KEY | Yes | — |
| GEMINI_MODEL | No | gemini-flash-latest |
| APP_KEY | Yes | (generated by key:generate) |
| DB_CONNECTION | No | sqlite |

## Endpoints

- `GET /health` → `{"status":"ok"}`
- `POST /analyze-ticket` → Structured JSON analysis
